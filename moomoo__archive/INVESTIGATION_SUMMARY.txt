  - User sees: NO ACTIVITY on Moomoo dashboard

BLOCKING ISSUE #2: No Market Data Subscriptions (CRITICAL)
----------------------------------------------------------
Location: nautilus_trader/adapters/moomoo/data.py

Problem:
  MoomooDataClient._connect() method is incomplete:
  1. Creates OpenQuoteContext but never subscribes to Moomoo
  2. Never sets up quote/ticker handlers
  3. Never subscribes to market data feeds
  4. Never implements bar aggregation (Moomoo sends quotes, not bars)

Evidence:
  - Strategies never receive on_bar() calls
  - self.bars_a and self.bars_b remain empty (need 20+ bars to generate signals)
  - Log lines 142-143 in pairs_trading.py show "if len(self.bars_a) < 20: return"
  - But bars are never populated because data never flows in

Impact:
  - Strategies have NO market data to process
  - Even if execution worked, no signals would be generated
  - Win rate would be 0% (can't trade without data)

Fix: 
  1. Add Moomoo subscription calls in _connect()
  2. Set up quote/ticker handlers
  3. Implement bar aggregation from ticks


HIGH RISK ISSUE #3: Excessive Position Sizing
----------------------------------------------
Location: scripts/start_paper_trading_moomoo.py

Problems:
  1. Position sizing: 6-8% of account (professional std: 1-2%)
  2. Hardcoded account value: $100,000 (not queried from Moomoo)
  3. No portfolio heat tracking (total capital at risk)
  4. No correlation checks between positions
  5. Mocked portfolio snapshots (all values hardcoded)

  - Excessive loss would kill account quickly

  - Risk management framework not integrated

Fix:
  1. Reduce position_size_pct to 0.02 (2%) in both strategies
  2. Query actual account value from Moomoo instead of hardcoding
  3. Add portfolio heat calculation
  4. Implement correlation checks
